Примери за API от социални мрежи

Информация

СЪЩНОСТ

API (Application Programming Interface) / приложно програмен интерфейс представлява трето лице – изпълняващо заявката за първо лице и обратното. Позволява на потребителя да използва готови фунционалности или информация, вместо да ги създава сам, което спестява време и финанси. Но какво представляват приложно програмните интерфейси предоставени от социални мрежи?

Те могат да бъдат HTTP заявки, към които се изпращат параметри и API ключ и според дадените параметри връщат JSON обект с информация, която може да се представя, манипулира и променя в зависимост от типа и HTTP заявката.

ПРИЛОЖЕНИЯ

Приложно програмните интерфейси имат много приложения, които зависят от това как потребителие ще ги използват или за каква цел ще ги използват. Най-често това бива автоматизиране на инфромация в уеб сайт или апликация, предоставяне на съдъражние за потребителите от дадената социална мрежа, улесняване на взаимодействието със социалната мрежа, което да бъде само от 1 място.

Пример: Получаване на видео клипове от YouTube канал в собствен уеб сайт.

ВЗАИМОДЕЙСТВИЕ

Информацията, която се връща към потребителя, идва от база данни, но взаимодействието не е директно, а чрез дадения API, който извършва самата комуникация, следователно осигуряващ улеснение за потребителя и по-висока сигурност за базата данни, особено ако става въпрос за социална мрежа, където запазването на информацията е изключително важно.

ИЗПОЛЗВАНЕ

Осъществява се GET заявка, с параметри като какъв тип данни се изискват, за кой акаунт и нужния API ключ. При успешно изпълнение, потребителя получава данните, с които може да извърши още една заявка, чрез успешното изпълнение на която да изобрази съдържанието й.

Пример: Кодова част за YouTube API
